# Biolab 数据采集任务台账

本文档只维护数据采集任务、批次、episode 状态和后续处理进度；流程说明见 `docs/biolab_data_pipeline.md`。

## 状态定义

| 状态 | 含义 | 进入下一阶段的条件 |
| --- | --- | --- |
| `planned` | 已规划，未采集 | 明确任务 step、起始条件、目标数量 |
| `collected` | 原始视频和 robot CSV 已采集 | 原始文件成组存在，坏数据已标记 |
| `preprocessed` | 已生成 `processed_dataset/<SESSION_ID>/` | `aligned_data.csv` 和相机图片已输出 |
| `verified` | 已完成基础校验 | 每个相机目录 jpg 数量等于 CSV 行数，曲线图可检查 |
| `ready_for_zarr` | 可进入 DexGraspVLA 数据构建 | 相机语义映射、state/action 维度、剔除列表已确认 |
| `needs_mask_fix` | 标注需人工修正 | 人工审查发现 mask 不可用，需修正后重建 zarr |
| `training` | 已进入训练 | zarr 已构建并通过结构校验，训练进程已启动 |
| `trained` | 已完成训练 | 已产出可用于回放/推理验证的 checkpoint |
| `rejected` | 不进入训练 | 缺关键数据、错误操作、严重不同步或质量不可用 |

## 更新规则

1. 每次采集后先补 `批次总表`，再补对应批次的 episode 明细。
2. 坏数据只记录，不删除原始文件；训练侧通过台账和脚本黑名单排除。
3. `是否进训练` 记录实际进入训练的数据；人工抽检结论写入备注或质量状态。
4. 只有通过图片数量、CSV 行数、曲线图和人工动作质量检查后，状态才从 `verified` 改为 `ready_for_zarr`。
5. 如果重跑预处理改变了帧数，必须同步更新 `有效帧数` 和 episode 明细。
