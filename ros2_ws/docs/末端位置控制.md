# OpenArm 末端位置控制

直接指定末端执行器的目标位置(x,y,z)和姿态，**整个机械臂（所有关节）会自动规划路径并运动到该位置**。

**工作原理：**
- 使用MoveIt2的`move_group` action进行运动规划
- MoveIt2会计算逆运动学（IK），确定所有7个关节的目标角度
- 进行路径规划，生成平滑的关节轨迹
- 执行运动，整个机械臂协调运动，使末端执行器到达目标位置

## 前置条件

1. **安装MoveIt2**（如果未安装）
   ```bash
   sudo apt install ros-humble-moveit
   ```

2. **启动MoveIt2**
   ```bash
   # 使用假硬件（仿真模式，无需真实机械臂）
   ros2 launch openarm_bimanual_moveit_config demo.launch.py use_fake_hardware:=true
   
   # 使用真实硬件
   ros2 launch openarm_bimanual_moveit_config demo.launch.py use_fake_hardware:=false
   
   # 真实硬件推荐使用 forward_position_controller（更稳定）
   ros2 launch openarm_bimanual_moveit_config demo.launch.py \
       use_fake_hardware:=false \
       robot_controller:=forward_position_controller
   ```
   
   **真实硬件前置条件：**
   - CAN接口已配置为CAN-FD模式
   - 电机已初始化（参考 `调试说明.md`）
   - 电机已使能

## 使用方法

```bash
cd ~/ros2_ws/src/openarm_teleop/script

# 基本用法：指定位置（默认姿态）
python3 move_to_pose.py left 0.3 0.2 0.4

# 指定位置和yaw角度
python3 move_to_pose.py right 0.3 -0.2 0.4 --yaw 1.57

# 完整姿态控制
python3 move_to_pose.py left 0.3 0.2 0.4 --roll 0.0 --pitch 0.0 --yaw 1.57
```

## 参数说明

- `arm_side`: `left` 或 `right`
- `x, y, z`: 目标位置（米），相对于base frame
- `--roll, --pitch, --yaw`: 欧拉角（弧度，可选，默认0）

## 坐标系

- **Base Frame**: `openarm_link0`
- **End Effector**: `openarm_left_hand` 或 `openarm_right_hand`

## 注意事项

- 目标位置必须在机械臂工作空间内
- 如果规划失败，可能是目标位置不可达或与障碍物碰撞
- 建议先用RViz可视化确认目标位置
- **从零位开始**：如果所有关节都在0位置，建议先尝试较小的目标位置，例如：
  ```bash
  # 从零位尝试较小的位移
  python3 move_to_pose.py left 0.2 0.1 0.3
  python3 move_to_pose.py right 0.2 -0.1 0.3
  ```

## 常见错误

### CONTROL_FAILED (-4)
如果出现 `CONTROL_FAILED` 错误，表示路径规划成功但执行失败。

**真实硬件排查：**
```bash
# 1. 检查控制器状态
ros2 control list_controllers

# 2. 检查CAN总线通信
candump can0  # 查看can0总线数据
candump can1  # 查看can1总线数据

# 3. 检查电机是否使能（查看启动日志）

# 4. 确认硬件已正确初始化
# 参考 调试说明.md 中的初始化步骤

# 5. 尝试使用 forward_position_controller
ros2 launch openarm_bimanual_moveit_config demo.launch.py \
    use_fake_hardware:=false \
    robot_controller:=forward_position_controller
```

**假硬件排查：**
- 检查MoveIt2启动终端的详细错误日志
- 尝试使用 `forward_position_controller` 而不是 `joint_trajectory_controller`

### 多个Action Server警告
如果看到 "There may be more than one action server" 警告，这是正常的，不影响功能。
