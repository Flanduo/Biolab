# 轨迹播放精度修复说明

录制 90° 播放时约 86° 的成因与对应修改建议。

---

## 原因简述

- **录制**：保存的是 `/joint_states` 的**实际关节位置**（反馈值）。
- **播放**：把相同数值作为**目标位置**发到 `forward_position_controller/commands`，由控制器按周期跟踪。

若两点间隔过短或最后一帧只发一次就结束，机械臂会一直“追不上”，最终停在比目标略小的角度。

---

## 建议修改

### 1. 播放结束时多“稳”一会儿最后一帧（最重要）

| 项目 | 说明 |
|------|------|
| **文件** | `play_joint_trajectory.py` |
| **位置** | `play_trajectory()` 里，播完 `for i in range(start_idx, len(self.trajectory_data))` 之后、在 `if self.loop and self.playing` 之前。 |

**改法：**  
播完所有点后，不要只等 0.1s 就结束，而是在 **1～2 秒内按固定频率（如 100Hz）反复发布最后一帧**的关节位置，再退出或进入循环，让机械臂有时间跟到目标角度并稳住。

---

### 2. 最后一点的“保持时间”加长并可调

| 项目 | 说明 |
|------|------|
| **文件** | `play_joint_trajectory.py` |
| **位置** | 同一段逻辑里，最后一个点的 `dt` 目前是 `0.1 / self.speed_factor`。 |

**改法：**

- 将末端保持时间改为 **1.0 秒**（或 0.5～2.0 秒可调），例如 `settle_time = 1.0`。
- 在这段时间内**按固定间隔（如 0.01s）多次发布最后一帧**，而不是只发一次再 sleep 一次，使目标角度被持续发送，机械臂更容易跟到 90° 而不是停在 86°。

---

### 3. 可选：不要用 min_interval 把 dt 强行放大

| 项目 | 说明 |
|------|------|
| **文件** | `play_joint_trajectory.py` |
| **位置** | `dt = max(dt, self.min_interval)` |

**改法：**

- 若录制频率不高（例如 ≤100Hz），可**不**把 `dt` 限制为至少 `min_interval`，直接按录制时的 `dt` 等待，时间线更接近录制。
- 若录制频率很高（例如 200Hz），可保留 `min_interval` 作上限，但将 `min_interval` 调小（例如 0.005，即 200Hz），避免播放节奏被压得过快、机械臂跟不上。

---

### 4. 可选：增加“播放减速”说明

| 项目 | 说明 |
|------|------|
| **文件** | `play_joint_trajectory.py` |

**改法：**  
已有 `--speed`，默认 1.0。在文档或 `--help` 中说明：若发现播放角度略小（如 90° 播成 86°），可尝试 `--speed 0.8` 或 `0.7`，让整条轨迹变慢，末端角度更接近录制。

---

### 5. 录制侧（一般不用改）

| 项目 | 说明 |
|------|------|
| **文件** | `record_joint_trajectory.py` |

录制存的是 `joint_states` 的 `position`，数值本身没问题。若希望“录命令而不是反馈”，可再开选项录 `/left_forward_position_controller/commands` 等；当前 90°→86° 现象主要是**播放节奏和末端稳住时间**问题，优先改播放即可。

---

## 总结

| 优先级 | 内容 |
|--------|------|
| **必做** | 在 `play_joint_trajectory.py` 里，播完轨迹后用 **1～2 秒反复发布最后一帧**，再结束/循环。 |
| **推荐** | 末端保持时间改为 1.0s（或可调），并在这段时间内按固定频率多次发布最后一帧。 |
| **可选** | 去掉或放宽 `dt = max(dt, self.min_interval)`，并建议用户用 `--speed 0.8` 做减速播放测试。 |

按上述修改后，播放角度应更接近录制（例如 90° 会跟到接近 90° 而不是 86°）。
