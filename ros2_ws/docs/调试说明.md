#### 新版初始化

**推荐：分步执行，不要一上来跑全流程。**  
`init_motors.sh` 按子命令拆开，每一步做完再确认，再执行下一步。

| 顺序 | 子命令 | 作用 |
|------|--------|------|
| 1 | `can20` | 把 CAN 配成 **2.0（1Mbps）**，写波特率前必须 |
| 2 | `baudrate` | 把电机波特率写成 **5Mbps** 并 `--flash`（有写入次数限制） |
| 3 | `zero` | 设零位（需先把关节摆到机械零位） |
| 4 | `canfd` | 把 CAN 配成 **CAN-FD**（与电机 5M 数据段匹配，日常运行用） |
| 5 | `verify` | 发使能帧，配合 `candump` 看是否回包 |

**常用命令（复制即用）：**

```bash
cd ~/ros2_ws
chmod +x init_motors.sh

# 分步（推荐）
./init_motors.sh can20
./init_motors.sh baudrate
./init_motors.sh zero
./init_motors.sh canfd
./init_motors.sh verify

# 帮助
./init_motors.sh help
```

**单电机（须同时写 --can 与 --id）：**

```bash
./init_motors.sh can20 --can can1 --id 4
./init_motors.sh baudrate --can can1 --id 4
./init_motors.sh zero --can can1 --id 4
./init_motors.sh canfd --can can1 --id 4
./init_motors.sh verify --can can1 --id 4
```

**仍要一键跑完全部时（兼容旧习惯）：**

```bash
./init_motors.sh all

# 或省略子命令，不写时默认就是 all
./init_motors.sh

# 跳过零位 / 跳过验证
./init_motors.sh all --skip-zero
./init_motors.sh all --skip-verify
```

**注意事项：**
- 写波特率前接口必须是 **CAN 2.0**；日常控制前接口应是 **CAN-FD**，顺序不要颠倒。
- 设置零位前会交互确认，务必已把电机摆到所需机械零位。
- 电机 Flash 写入次数有限，不要频繁执行 `baudrate`。

#### 旧版初始化

#### 设置接口波特率

**使用说明：**
- 以下所有命令都可以直接复制粘贴到终端执行
- 多行命令（包含 `for ... do ... done` 的循环）需要完整复制整个代码块
- 如果使用 `sh` 而不是 `bash`，可能需要将 `{1..8}` 替换为 `1 2 3 4 5 6 7 8`
- 确保在正确的目录下执行命令（按提示先 `cd` 到相应目录）

##### 选项 1：使用canfd脚本一键激活

```bash
cd ~/ros2_ws

# 循环配置所有 CAN 接口为 CAN-FD 模式
for can_if in can0 can1; do
    sudo ./src/openarm_can/setup/configure_socketcan.sh $can_if -fd
done
```


##### 选项 2：手动设置
**CAN 2.0 模式**

```bash
# 循环配置所有 CAN 接口为 CAN 2.0 模式
for can_if in can0 can1; do
    sudo ip link set $can_if down
    sudo ip link set $can_if type can bitrate 1000000
    sudo ip link set $can_if up
done
```

**CAN FD 模式**

```bash
# 循环配置所有 CAN 接口为 CAN-FD 模式
for can_if in can0 can1; do
    sudo ip link set $can_if down
    sudo ip link set $can_if type can bitrate 1000000 dbitrate 5000000 fd on
    sudo ip link set $can_if up
done
```

##### 选项 3：使用安装脚本

OpenArm CAN 库包括方便的设置脚本，示例用法：

```bash
# Navigate to openarm_can directory
git clone https://github.com/enactic/openarm_can.git
cd openarm_can/setup

# For CAN 2.0
./configure_socketcan.sh can0

# For CAN FD at 5mbps (recommended)
./configure_socketcan.sh can0 -fd -b 1000000 -d 5000000

sudo ./src/openarm_can/setup/configure_socketcan.sh can0 -fd
```

# 检查所有 CAN 接口状态
for can_if in can0 can1; do

#### 电机测试

该测试将使用简单的 CAN 命令验证与电机的基本通信。

**步骤 1 - 监控 CAN 流量：** 打开两个终端窗口分别监控

```bash
# 终端 1：监控 can0
candump can0

# 终端 2：监控 can1
candump can1
```

保持此运行以观察所有 CAN 消息。

**步骤 2 - 发送命令：** 在终端 3 中发送测试命令

```bash
# 向所有总线上的所有电机（ID 1-8）发送使能命令
# 注意：如果使用 sh 而不是 bash，将 {1..8} 替换为 1 2 3 4 5 6 7 8
for can_if in can0 can1; do
    for motor_id in {1..8}; do
        can_id=$(printf "%03d" $motor_id)
        echo "使能 $can_if 总线电机 ID $motor_id"
        cansend $can_if ${can_id}#FFFFFFFFFFFFFFFC
    done
done

# 或者使用兼容性更好的方式（适用于所有 shell）：
for can_if in can0 can1; do
    for motor_id in 1 2 3 4 5 6 7 8; do
        can_id=$(printf "%03d" $motor_id)
        echo "使能 $can_if 总线电机 ID $motor_id"
        cansend $can_if ${can_id}#FFFFFFFFFFFFFFFC
    done
done

# 向电机 #1 发送失能命令（示例）
cansend can0 001#FFFFFFFFFFFFFFFD
```

**命令分解：**

- `001` = 电机 #1 CAN ID （0x01）
- `FFFFFFFFFFFFFFFC` = 电机使能命令
- `FFFFFFFFFFFFFFFD` = 电机失能命令

#### 设置电机波特率

* ==无论您的目标波特率如何，请务必在波特率设置期间使用 2.0 上的 CAN 接口。完成此步骤后，将其配置回目标波特率==。

- 电机参数有 10000 次写入时间限制，不应在脚本中频繁运行。

**批量设置所有电机（推荐）：**

```bash
# 进入 setup 目录
cd ~/ros2_ws/src/openarm_can/setup

# 为所有总线上的所有电机（ID 1-8）设置波特率
# 注意：如果使用 sh 而不是 bash，将 {1..8} 替换为 1 2 3 4 5 6 7 8
for can_if in can0 can1; do
    for motor_id in {1..8}; do
        echo "设置 $can_if 总线电机 ID $motor_id..."
        python3 change_baudrate.py --baudrate 5000000 --canid $motor_id --socketcan $can_if --flash
        sleep 0.5  # 短暂延迟，避免连续写入过快
    done
done

# 或者使用兼容性更好的方式（适用于所有 shell）：
for can_if in can0 can1; do
    for motor_id in 1 2 3 4 5 6 7 8; do
        echo "设置 $can_if 总线电机 ID $motor_id..."
        python3 change_baudrate.py --baudrate 5000000 --canid $motor_id --socketcan $can_if --flash
        sleep 0.5  # 短暂延迟，避免连续写入过快
    done
done
```

**重要：**完成波特率配置后，重新配置 CAN 接口以匹配电机的新波特率设置，以便在必要时进行适当的通信。

#### 设置零位

* 警告：在设置电机之前，请将电机物理定位到所需的零位置，尤其是在您已经组装了手臂的情况下！对于夹持器，默认情况下，闭合位置用作零。

**设置特定电机的零位：**

```bash
./set_zero.sh can0 001
```

**为单个总线上的所有电机设置零位：**

```bash
# can0 总线
./set_zero.sh can0 --all

# can1 总线
./set_zero.sh can1 --all
```

**为所有总线上的所有电机设置零位：**

```bash
cd ~/ros2_ws/src/openarm_can/setup

for can_if in can0 can1; do
    echo "设置 $can_if 总线上所有电机的零位..."
    ./set_zero.sh $can_if --all
done
```

#### 验证电机通信

运行电机检查可执行文件进行通信验证。可执行文件应该在库编译的构建目录中可用。如果缺少，请重新生成项目。

* 注意：CAN ID 17 是 0x11 的十进制表示形式（发送 ID 1，接收 ID 17）
* 一定注意使用==对应的模式==测试

**CAN 2.0 模式测试：**

```bash
cd ~/ros2_ws/src/openarm_can/build

# 测试单个总线
./motor-check 1 17 can0

# 测试所有总线
for can_if in can0 can1; do
    echo "测试 $can_if 总线..."
    ./motor-check 1 17 $can_if
done
```

**CAN FD 模式测试：**

```bash
cd ~/ros2_ws/src/openarm_can/build

# 测试单个总线
./motor-check 1 17 can0 -fd

# 测试所有总线
for can_if in can0 can1; do
    echo "测试 $can_if 总线..."
    ./motor-check 1 17 $can_if -fd
done
```

输出应显示电机的接收器 ID、波特率和状态信息。
