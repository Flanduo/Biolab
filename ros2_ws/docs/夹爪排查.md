# 夹爪无响应排查（简洁版）

## 1. 先确认控制器是否在线

```bash
ros2 control list_controllers
```

期望至少看到以下控制器为 `active`：
- `joint_state_broadcaster`
- `left_gripper_controller`（左夹爪）
- `right_gripper_controller`（右夹爪）

如果未激活，先启动双臂 bringup：

```bash
ros2 launch openarm_bringup openarm.bimanual.launch.py arm_type:=v10
```

---

## 2. 检查 action 接口是否存在

```bash
ros2 action list | findstr gripper_cmd
```

期望看到：
- `/left_gripper_controller/gripper_cmd`
- `/right_gripper_controller/gripper_cmd`

若没有，说明夹爪控制器没启动成功（回到步骤 1）。

---

## 3. 直接发 action 测试（最小闭环）

### 左夹爪张开到 0.04

```bash
ros2 action send_goal /left_gripper_controller/gripper_cmd control_msgs/action/GripperCommand "{command: {position: 0.04, max_effort: 40.0}}"
```

### 左夹爪闭合到 0.0

```bash
ros2 action send_goal /left_gripper_controller/gripper_cmd control_msgs/action/GripperCommand "{command: {position: 0.0, max_effort: 40.0}}"
```

右夹爪同理，把 `left` 改为 `right`。

---

## 4. 看关节状态是否有变化

```bash
ros2 topic echo /joint_states
```

关注：
- `openarm_left_finger_joint1`
- `openarm_right_finger_joint1`

发目标后关节值若完全不变：
- 控制链路可能未接通（控制器/硬件接口问题）
- 或 CAN 通讯异常

---

## 5. 使用项目内脚本测试（推荐）

项目已提供脚本：

```bash
python3 src/openarm_teleop/script/control_gripper.py left 0.02 20 0.8 16
```

参数含义：
- `left/right/both`：控制哪侧
- `position`：`0.0` 闭合，`0.04` 张开
- `max_effort`：力度上限
- `duration`：总时长（秒）
- `steps`：插值步数

---

## 6. 常见根因速查

- **控制器名字用错**：本项目是 `left_gripper_controller` / `right_gripper_controller`，不是 `gripper_controller`。
- **只启动了单臂**：会导致另一侧 action 不存在。
- **`use_fake_hardware:=true`**：仿真可动但真机不动（确认当前启动参数）。
- **CAN 接口异常**：机械臂关节也常伴随无响应，需先恢复总线通讯。

---

## 7. 一条命令快速自检（左夹爪）

```bash
ros2 control list_controllers && ros2 action list | findstr left_gripper_controller && ros2 action send_goal /left_gripper_controller/gripper_cmd control_msgs/action/GripperCommand "{command: {position: 0.02, max_effort: 30.0}}"
```
